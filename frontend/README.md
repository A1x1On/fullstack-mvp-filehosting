# Frontend — File Manager UI

Next.js application for managing uploaded files and viewing security scan alerts. Communicates with the FastAPI backend over REST.

## Stack

| Layer | Technology |
|-------|-----------|
| Framework | Next.js (App Router), React 18 |
| State / data fetching | Redux Toolkit + RTK Query |
| UI | Bootstrap 5 + react-bootstrap, Sass modules |
| File download | file-saver |
| Language | TypeScript 5 |
| Testing | Vitest + Testing Library |

## Setup (local)

```bash
# 1. Install dependencies
npm install
# or
yarn install

# 2. Configure environment
# Create a .env.local file (or set env vars) — see Environment variables section below

# 3. Start development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Setup (Docker Compose)

```bash

docker compose -f docker-compose.dev.yml up --build
```

The frontend container is built with `output: standalone` and served on port **3000**.

## Features

### File management

- **Upload** — form with a title field and file picker; sends `multipart/form-data` to the backend.
- **List** — paginated table showing title, original filename, MIME type, size, processing status, scan status, and upload date.
- **Download** — fetches the file as a blob and saves it to disk via `file-saver`.
- **Status badges** — Bootstrap colour-coded badges for `processingStatus` (pending / processing / done / failed) and `scanStatus` (safe / warning / critical / pending).

### Alerts viewer

- Paginated table of security alerts generated by the backend Celery pipeline.
- Each row shows alert ID, linked file ID, severity level badge, message, and date.

### Pagination

- Shared `RTable` component handles rendering, loading state, empty state, and Bootstrap pagination controls.
- Pagination state (limit / offset) is stored in Redux slices; changing the page dispatches a new `getByCriteria` query.

## Environment variables

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_BACKEND_ENDPOINT` | Backend base URL in **development** (e.g. `http://localhost:8000`) |
| `NEXT_PUBLIC_BACKEND` | Backend base URL in **production** |
| `NEXT_PUBLIC_BASE_PATH` | Optional Next.js `basePath` (default: `''`) |

Create a `.env.local` file at the root of the `frontend/` directory:

```env
NEXT_PUBLIC_BACKEND_ENDPOINT=http://localhost:8000
```

## Running tests

- `FilesHeader` upload form behaviour

- `general.ts` utility functions (`formatDate`, `formatSize`)
